Day12-設計基本路由

前言

延續，我們設定當造訪根目錄 ‘/‘時，會回傳 Hello World，接著可以此類推，設定當造訪其他頁面的路由（router）

Express 定義路由的結構如下

  • app 是 express 的實例。
  • METHOD 是 HTTP 要求方法。
  • PATH 是伺服器上的路徑。
  • HANDLER 是當路由相符時要執行的函數。

以最終作業設計稿為例，當造訪「全體動態牆」頁面時，路由可以設定為

新增貼文（POST 請求）

若是使用者的個人頁面，因每個使用者會有各自的 id 或帳號，此時就可以設計成動態路由
再透過 req.params 取得參數

練習

設計當使用者造訪以下頁面（GET）時的路由， response 可先回傳一段簡單的文字即可

  • 登入
  • 註冊
  • 全體動態牆
  • 個人牆
  • 個人追蹤名單
